# src/cloud — Lumina-Cloud integration (vendored)

Эти файлы **вендорены** (скопированы) из бэкенд-репозитория **Lumina-Cloud**. Не редактировать
вручную — они генерируются там и обновляются копированием.

- `contracts.cjs` — самодостаточный бандл контрактов API `/v1` (словари рейтингов/статусов,
  формы ответов каталога/загрузки/auth, пути `API_PATHS`, лимиты `UPLOAD_LIMITS`). zod вшит внутрь
  — никаких новых npm-зависимостей. Подключение: `const C = require("./cloud/contracts.cjs");`.
- `contracts.d.ts` — типы для подсказок IDE.

## Обновление

В репозитории Lumina-Cloud: `npm run build -w packages/contracts`, затем скопировать
`packages/contracts/dist/lumina-contracts.cjs` → `contracts.cjs` и `.d.ts` → `contracts.d.ts`.
Внутри `/v1` бэкенд меняется только аддитивно, так что старый бандл не ломается на новом сервере.

## Статус встройки

Сейчас это **только фундамент** (задача 4.8b): файлы лежат, но к живому коду приложения **ещё не
подключены**. Полная инструкция по встройке (вход через Google, эндпоинты, CSP, порядок) —
в Lumina-Cloud: `docs/client-integration.md`. Делать встройку отдельными сессиями по ритуалу
этого репозитория, начиная с тонкого тестируемого модуля-клиента, не вкручивая сразу в `main.js`.
